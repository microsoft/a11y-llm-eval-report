Expand/Collapse Widget Examples

Example 1: Basic Information

An expand/collapse widget is an interactive component that allows users to show or hide content on demand. This helps keep interfaces clean and organized while still providing access to detailed information when needed.

These widgets are commonly used in FAQs, navigation menus, and content-heavy pages to improve readability and user experience.

Example 2: Accessibility Features

This widget follows WCAG 2.2 Level AA guidelines and includes:

  • Semantic HTML with proper button elements
  • ARIA attributes (aria-expanded and aria-controls) to communicate state
  • Full keyboard support (Space and Enter keys to toggle)
  • Clear focus indicators for keyboard navigation
  • Sufficient color contrast ratios
  • Descriptive button text that explains the action

Example 3: Technical Details

The widget uses vanilla JavaScript to handle interactions and update ARIA states. The button element is used for proper semantics, and the hidden attribute controls visibility of the content.

CSS transitions provide smooth visual feedback, and the focus outline ensures keyboard users can easily see which element is active.

Each widget instance is independent and can be controlled separately, making it easy to use multiple widgets on the same page.

Example 4: Use Cases

Expand/collapse widgets are versatile and can be used in many contexts:

  • FAQ sections: Allow users to browse questions and expand only the ones they're interested in
  • Product descriptions: Show basic info by default with detailed specifications available on demand
  • Form sections: Break long forms into manageable chunks
  • Navigation menus: Create hierarchical menu structures
  • Settings panels: Organize advanced options without overwhelming users
