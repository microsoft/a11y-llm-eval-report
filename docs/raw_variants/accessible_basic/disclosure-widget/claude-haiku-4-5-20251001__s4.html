Expand/Collapse Widget Examples

Example 1: Native HTML <details> Element

Click to expand

This is a native HTML <details> element. It provides built-in expand/collapse functionality with full keyboard support and semantic meaning.

The <summary> element acts as the toggle button, and the content below it is automatically hidden or shown.

  • Fully accessible by default
  • Keyboard navigable
  • Screen reader friendly
  • No JavaScript required

Example 2: Multiple Expandable Sections

Section 1: Features

You can have multiple <details> elements on the same page. Each one operates independently.

  • Expand one section
  • Expand multiple sections
  • Collapse any section
Section 2: Benefits

Using native HTML elements provides several benefits:

  • Better performance
  • Improved accessibility
  • Reduced JavaScript complexity
  • Better browser support
Section 3: Usage

The <details> element is perfect for FAQs, documentation, and any content that benefits from progressive disclosure.

Users can control what information they see, improving the overall user experience.

Example 3: Custom Expand/Collapse Widget

This is a custom expand/collapse widget built with JavaScript. It uses ARIA attributes for accessibility.

The aria-expanded attribute indicates whether the content is visible, and aria-controls links the button to the content it controls.

  • Custom styling and animations
  • Full keyboard support
  • ARIA attributes for screen readers
  • Smooth transitions

Example 4: Pre-Expanded Section

This section is open by default

You can use the open attribute on the <details> element to have a section expanded when the page loads.

This is useful for highlighting important information or the most commonly accessed content.
