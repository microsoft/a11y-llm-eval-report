Expand/Collapse Widget Demo

Project Overview

This project explores the development of accessible web components. The expand/collapse widget is a common UI pattern used to manage content visibility while maintaining a clean layout.

Key considerations include keyboard navigation, screen reader announcements, and smooth visual transitions. The content area is programmatically associated with the button using ARIA attributes.

Technical Specifications

Framework: Vanilla HTML, CSS, JavaScript

Accessibility: WCAG 2.2 AA compliant. Implements proper focus management, ARIA states, and semantic markup.

Features: The widget uses CSS transitions for smooth expansion and collapse. The button text updates dynamically to reflect the current state.

Browser Support: Compatible with all modern browsers. Gracefully degrades if JavaScript is disabled.

Implementation Notes

The widget is built using a button element to ensure native keyboard interactivity (Enter and Space keys). The aria-expanded attribute is toggled between true and false to inform assistive technologies of the content's state.

The aria-controls attribute creates an explicit relationship between the button and the content container. A visually hidden span provides additional context for screen reader users about the initial state.

CSS handles the visual expansion/collapse via the max-height and opacity properties, triggered by adding/removing an expanded class.
