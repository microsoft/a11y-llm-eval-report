Expand/Collapse Widget

The following examples demonstrate accessible disclosure (expand/collapse) widgets. Each section can be toggled open or closed using a button. Use Enter or Space to activate the button.

Examples

An expand/collapse widget, also known as a disclosure widget, allows users to show or hide a section of content. It consists of a toggle button and a controlled content region. When the button is activated, the content is revealed; activating it again hides the content.

This pattern is useful for reducing visual clutter while keeping information readily available on demand.

This widget includes the following accessibility features:

  • The toggle uses a native <button> element, ensuring keyboard operability.
  • aria-expanded communicates the current state (open or closed) to assistive technology.
  • aria-controls identifies the content region controlled by the button.
  • The content region uses role="region" with aria-labelledby pointing to the button, providing a labeled landmark when expanded.
  • The hidden attribute ensures collapsed content is not accessible to any user.
  • Focus indicators are clearly visible and meet contrast requirements.

Consider using an expand/collapse widget when:

  • You have supplementary content that not all users need to see immediately.
  • A page has many sections and you want to reduce scrolling while keeping content available.
  • You are building a frequently asked questions (FAQ) section.
  • You want to progressively disclose information to reduce cognitive load.

Avoid using this pattern for content that is essential for all users to see on page load, such as primary calls to action or critical instructions.
