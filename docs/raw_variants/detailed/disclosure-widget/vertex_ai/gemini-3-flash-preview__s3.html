The following examples demonstrate accessible patterns for toggling the visibility of content.

1. Custom Disclosure Pattern

Using a standard <button> with aria-expanded and aria-controls.

Web accessibility (also referred to as a11y) is the design and creation of websites that can be used by everyone. This includes people with visual, auditory, physical, speech, cognitive, and neurological disabilities.

2. Native HTML Details Pattern

Using the native <details> and <summary> elements for built-in accessibility.

How do I improve my site's accessibility?

You can start by following the Web Content Accessibility Guidelines (WCAG). Key steps include:

  • Using semantic HTML elements.
  • Ensuring high color contrast.
  • Providing text alternatives for non-text content.
  • Making all functionality available via keyboard.
