Expand/Collapse Widget Examples

These widgets demonstrate accessible expandable sections using native HTML and ARIA attributes.

Example 1: Frequently Asked Question

This demo is built to conform to WCAG 2.2 Level AA guidelines. It includes proper keyboard navigation, sufficient color contrast, semantic HTML, and support for assistive technologies.

All interactive elements are focusable, have visible labels, and provide clear feedback. The code avoids unnecessary ARIA and prefers native HTML semantics where possible.

Example 2: Technical Details

The widget uses a <button> element with aria-expanded to communicate state and aria-controls to associate it with the content region.

The content container uses the hidden attribute for simple show/hide behavior. JavaScript toggles the attribute and updates aria-expanded.

Focus management is handled naturally by the button. The Tab key moves to the button; Enter or Space activates it. The content that follows is in DOM order.

  • Color contrast meets 4.5:1 for text.
  • Focus indicator has a 3:1 contrast ratio.
  • The layout reflows to 320px without horizontal scrolling.

Example 3: Additional Instructions

When using a screen reader:

  1. Navigate to the button using Tab or virtual cursor commands.
  2. The button will announce its role, name, and current state (e.g., "collapsed").
  3. Press Enter or Space to toggle the content.
  4. The updated state (e.g., "expanded") will be announced, and you can navigate into the newly revealed content.

The decorative arrow icon is hidden from assistive technology with aria-hidden="true" because the state is communicated via aria-expanded.

This demo is for educational purposes. Always test with real users and assistive technology.
