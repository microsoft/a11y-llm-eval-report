Click to expand: What is accessibility?

Web accessibility means designing and developing websites and applications that are usable by everyone, including people with disabilities. This includes people with visual, auditory, motor, and cognitive disabilities.

Accessible design benefits all users by making content clearer, navigation easier, and interactions more intuitive.

Click to expand: Why is it important?

Accessibility is important because:

  • It ensures equal access to information and services
  • It improves usability for everyone, not just people with disabilities
  • It is often a legal requirement in many jurisdictions
  • It demonstrates respect and inclusion for all users

WCAG 2.2 is built on four core principles:

  1. Perceivable: Information must be presentable to users in ways they can perceive
  2. Operable: Interface components must be operable via keyboard and other input methods
  3. Understandable: Information and user interface operation must be understandable
  4. Robust: Content must be robust enough to be interpreted by a wide variety of user agents and assistive technologies

Keyboard navigation is essential for accessibility. Users should be able to:

  • Tab through all interactive elements in a logical order
  • Use Enter or Space to activate buttons
  • Use Escape to close dialogs or menus
  • Use arrow keys to navigate within composite widgets

Multiple expandable sections that can be opened independently. Each section maintains its own state.

Section 1: Color Contrast

Color contrast is critical for readability. WCAG 2.2 requires:

  • Normal text: 4.5:1 contrast ratio
  • Large text (18pt+): 3:1 contrast ratio
  • UI components and focus indicators: 3:1 contrast ratio
Section 2: Focus Management

Focus management ensures users know where they are on the page:

  • Focus indicators must be visible (at least 2px)
  • Focus order should follow logical reading order
  • Focus should not be trapped on the page
  • When opening dialogs, focus should move to the dialog
Section 3: Semantic HTML

Using semantic HTML improves accessibility:

  • Use <button> for buttons, not <div>
  • Use <nav> for navigation regions
  • Use <main> for main content
  • Use heading tags (<h1>-<h6>) to structure content
