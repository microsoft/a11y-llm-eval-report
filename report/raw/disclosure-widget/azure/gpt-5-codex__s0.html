Explore native and custom implementations of expand/collapse interactions, each built to be accessible, responsive, and easy to integrate.

Native HTML Details

A semantic <details> element provides built-in expand/collapse behavior, including keyboard support and ARIA states.

How does the native widget work?

The browser handles all expand/collapse logic and accessibility when you use the <details> element paired with a <summary>. It automatically toggles open and closed states, updates focus targets, and communicates state changes to assistive technologies.

  • Keyboard users can press Enter or Space to toggle.
  • Screen readers announce whether the section is expanded.
  • Developers can listen for the toggle event for custom hooks.

Custom Toggle Button

A button controls the visibility of a dedicated content container. ARIA attributes ensure the relationship is conveyed to assistive tech.
