An expand/collapse widget reveals supplemental information on demand, helping declutter interfaces while keeping useful details accessible.

This approach is especially useful for FAQ sections, documentation, or any scenario where users should control when they see extra content.

Code Snippet Explanation

// Example pseudo-code
function expandSection(button, content) {
  const isOpen = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!isOpen));
  content.hidden = isOpen;
}

The widget updates the aria-expanded state for accessibility, while the hidden attribute toggles visibility. This preserves keyboard navigation and screen reader support.
