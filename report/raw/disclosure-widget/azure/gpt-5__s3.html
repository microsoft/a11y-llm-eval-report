Expand/Collapse Widget Examples

Three approaches: native details/summary, an accessible disclosure button, and a CSS-only version. Each widget is wrapped in an element with class "example", and the container for controlled content uses class "details".

1) Native details/summary
▸ What is an expand/collapse widget?

Expand/collapse toggles visibility of related content. The native <details> element provides built‑in accessibility and keyboard support.

  • Keyboard accessible by default
  • Declarative markup with <summary> as the control
  • Works without JavaScript
<details class="details">
  <summary>Question</summary>
  <p>Answer</p>
</details>
2) Accessible disclosure button

This version uses a button with aria-expanded and aria-controls to create an accessible disclosure pattern.

  • Explicit ARIA relationships
  • Works with screen readers and keyboard navigation
  • Fully controllable via JavaScript
<button aria-expanded="false" aria-controls="content">Toggle</button>
<div id="content" class="details" hidden role="region" aria-labelledby="button-id">...</div>
3) CSS-only disclosure

This approach uses a hidden checkbox and a label. When checked, sibling CSS selectors reveal the content.

  • No JavaScript required
  • Relies on semantic form controls
  • Useful for simple collapsible sections
input:checked + label + .details { display: block; }
